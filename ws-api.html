WebSocket 聊天室接口文档

概述

WebSocket 接口提供实时双向通信能力，支持用户加入房间、发送消息、管理用户等功能。

连接信息

WebSocket URL

ws://localhost:8080/ws/chat

连接参数

Token 认证

  • token: JWT Token（通过URL参数传递）

    • 类型：String

    • 必填：是

    • 说明：通过登录接口获取的JWT Token，用于身份认证

    • 格式：Bearer Token，通过URL参数传递

    • 示例：ws://localhost:8080/ws/chat?token=eyJhbGciOiJIUzI1NiJ9...

    • 注意：Token默认有效期为7天，过期后需要重新登录

消息类型

1. JOIN - 加入房间

功能说明

将当前用户加入指定的聊天房间。加入房间后会收到最近10条历史消息，房间内其他用户会收到系统通知。

请求参数

参数详细说明

type

  • 固定值："JOIN"

  • 说明：标识此消息为加入房间请求

  • 必填：是

roomId

  • 类型：String

  • 说明：要加入的房间ID

  • 必填：是

  • 格式：纯数字字符串

  • 长度：1-20个字符

  • 约束：房间必须存在且处于活跃状态

  • 获取方式：通过 ROOM_LIST 消息获取房间列表

password

  • 类型：String

  • 说明：房间密码

  • 必填：否（私有房间必填）

  • 约束：

    • 私有房间（isPrivate=true）：必须提供正确的密码

    • 公开房间（isPrivate=false）：无需提供密码

  • 长度：1-100个字符

  • 注意：密码错误时会返回错误消息

content（兼容字段）

  • 类型：String

  • 说明：旧版本客户端传递房间密码的字段

  • 必填：否

  • 注意：仅在未提供 password 时作为回退字段使用

响应参数

响应详细说明

type

  • 固定值："JOIN"

  • 说明：标识此响应为加入房间成功

content

  • 类型：String

  • 说明：加入房间的结果描述

  • 格式："成功加入房间: {房间名称}"

  • 示例："成功加入房间: 测试聊天室"

roomId

  • 类型：String

  • 说明：成功加入的房间ID

  • 与请求参数中的roomId一致

timestamp

  • 类型：Long

  • 说明：服务器处理请求的时间戳

  • 单位：毫秒

  • 格式：Unix时间戳

3. CHAT - 发送聊天消息

功能说明

向当前房间发送聊天消息。消息会经过敏感词检测，包含敏感词的消息会被拒绝。消息会广播给房间内所有用户（包括发送者）。

请求参数

参数详细说明

type

  • 固定值："CHAT"

  • 说明：标识此消息为聊天消息

  • 必填：是

content

  • 类型：String

  • 说明：要发送的聊天消息内容

  • 必填：是

  • 约束：

    • 不能为空

    • 不能包含敏感词（会被系统拒绝）

    • 长度：1-1000个字符

    • 支持中文、英文、数字、常用标点符号

  • 注意：包含敏感词的消息会被拒绝，并返回错误信息

响应参数

响应详细说明

messageId

  • 类型：String

  • 说明：消息的唯一标识符

  • 格式：纯数字字符串

  • 用途：用于撤回消息时指定要撤回的消息

  • 注意：每个消息都有唯一的messageId

isRecalled

  • 类型：Boolean

  • 说明：消息是否已被撤回

  • 可能值：true（已撤回）、false（未撤回）

  • 默认值：false

系统广播

消息会广播给房间内所有用户（包括发送者），所有用户收到的消息内容相同。

注意事项

  1. 敏感词检测：所有聊天消息都会经过敏感词检测，包含敏感词的消息会被拒绝

  2. 广播机制：消息会广播给房间内所有用户（包括发送者）

  3. 消息撤回：发送者可以在2分钟内撤回消息

  4. 禁言限制：被禁言的用户无法发送聊天消息

  5. 房间限制：必须先加入房间才能发送聊天消息

错误响应

4. CREATE_ROOM - 创建房间（管理员）

功能说明

创建新的聊天房间。仅管理员可用。创建成功后会自动发送 ROOM_LIST 消息更新房间列表。

请求参数

参数详细说明

type

  • 固定值："CREATE_ROOM"

  • 说明：标识此消息为创建房间请求

  • 必填：是

content

  • 类型：String

  • 说明：房间名称

  • 必填：是

  • 约束：

    • 不能为空

    • 长度：1-50个字符

    • 不能包含特殊字符

    • 房间名称在系统中必须唯一

  • 注意：房间名称已存在时会返回错误

description

  • 类型：String

  • 说明：房间描述

  • 必填：否

  • 约束：

    • 长度：0-200个字符

    • 支持中文、英文、数字、常用标点符号

  • 默认值：空字符串

maxUsers

  • 类型：Integer

  • 说明：房间最大用户数

  • 必填：否

  • 约束：

    • 范围：1-1000

    • 必须为整数

  • 默认值：50

  • 注意：当房间用户数达到上限时，新用户无法加入

isPrivate

  • 类型：Boolean

  • 说明：是否为私有房间

  • 必填：否

  • 可能值：true（私有）、false（公开）

  • 默认值：false

  • 约束：

    • true：需要密码才能加入

    • false：任何人都可以加入

password

  • 类型：String

  • 说明：房间密码

  • 必填：否（私有房间必填）

  • 约束：

    • 私有房间（isPrivate=true）：必须提供密码

    • 公开房间（isPrivate=false）：无需提供密码

    • 长度：1-50个字符

  • 注意：密码错误时用户无法加入房间

响应参数

注意事项

  1. 权限要求：仅管理员可用，非管理员用户会收到"权限不足"错误

  2. 自动更新：创建成功后会自动发送 ROOM_LIST 消息更新房间列表

  3. 房间名称：房间名称在系统中必须唯一

  4. 私有房间：私有房间需要设置密码，加入时需要提供正确的密码

错误响应

5. DELETE_ROOM - 删除房间（管理员）

功能说明

删除指定的聊天房间。仅管理员可用。删除成功后会自动发送 ROOM_LIST 消息更新房间列表。房间内所有用户会被移出房间。

请求参数

参数详细说明

type

  • 固定值："DELETE_ROOM"

  • 说明：标识此消息为删除房间请求

  • 必填：是

roomId

  • 类型：String

  • 说明：要删除的房间ID

  • 必填：是

  • 格式：纯数字字符串

  • 长度：1-20个字符

  • 约束：房间必须存在

  • 获取方式：通过 ROOM_LIST 消息获取房间列表

响应参数

系统广播

删除成功后：

  1. 自动发送 ROOM_LIST 消息更新房间列表

  2. 房间内所有用户会被移出房间

  3. 被移出的用户会收到系统通知

注意事项

  1. 权限要求：仅管理员可用

  2. 自动更新：删除成功后会自动发送 ROOM_LIST 消息

  3. 用户移出：房间内所有用户会被移出房间

  4. 不可恢复：删除的房间无法恢复

错误响应

响应详细说明

rooms

  • 类型：String

  • 格式：纯数字字符串

  • 用途：用于加入房间、删除房间等操作

name

  • 类型：String

  • 说明：房间名称

  • 长度：1-50个字符

description

  • 类型：String

  • 说明：房间描述

  • 长度：0-200个字符

  • 可以为空

maxUsers

  • 类型：Integer

  • 说明：房间最大用户数

  • 范围：1-1000

  • 约束：当userCount达到maxUsers时，新用户无法加入

isPrivate

  • 类型：Boolean

  • 说明：是否为私有房间

  • 可能值：

    • true：私有房间，需要密码才能加入

    • false：公开房间，任何人都可以加入

11. RECALL - 撤回消息

功能说明

撤回之前发送的消息。仅消息发送者可用，且消息发送时间不超过2分钟。撤回消息会广播给房间内所有用户。

请求参数

参数详细说明

type

  • 固定值："RECALL"

  • 说明：标识此消息为撤回消息请求

  • 必填：是

messageId

  • 类型：String

  • 说明：要撤回的消息ID

  • 必填：是

  • 格式：纯数字字符串

  • 长度：1-20个字符

  • 约束：

    • 消息必须存在

    • 必须是当前用户发送的消息

    • 消息发送时间不超过2分钟

  • 获取方式：从聊天消息响应中获取

响应参数

系统广播

撤回消息会广播给房间内所有用户：

注意事项

  1. 发送者限制：只能撤回自己发送的消息

  2. 时间限制：消息发送时间不超过2分钟

  3. 广播机制：撤回消息会广播给房间内所有用户

  4. 消息状态：撤回后，消息的isRecalled字段变为true

  5. 不可恢复：撤回的消息无法恢复

13. CHECK_SENSITIVE - 检测敏感词

功能说明

检测文本中是否包含敏感词，并返回检测到的敏感词列表和过滤后的文本。使用AC自动机算法进行检测。

请求参数

参数详细说明

type

  • 固定值："CHECK_SENSITIVE"

  • 说明：标识此消息为敏感词检测请求

  • 必填：是

content

  • 类型：String

  • 说明：要检测的文本内容

  • 必填：是

  • 约束：

    • 不能为空

    • 长度：1-1000个字符

    • 支持中文、英文、数字、常用标点符号

  • 注意：检测不区分大小写

响应参数

响应详细说明

contains

  • 类型：Boolean

  • 说明：文本是否包含敏感词

  • 可能值：

    • true：包含敏感词

    • false：不包含敏感词

detectedWords

  • 类型：Array

  • 说明：检测到的敏感词列表

  • 元素类型：String

  • 数组长度：0-N

  • 约束：

    • 不包含敏感词时为空数组

    • 包含多个敏感词时返回所有敏感词

  • 注意：返回的敏感词为小写形式

filteredText

  • 类型：String

  • 说明：过滤后的文本

  • 说明：敏感词被替换为*号

  • 示例：

    • 原文："这是敏感词测试"

    • 过滤后："这是***测试"

  • 注意：过滤不区分大小写

检测算法

系统使用AC自动机算法进行敏感词检测：

  • 时间复杂度：O(n)，n为文本长度

  • 特点：适用于大量敏感词的高效匹配

  • 优势：一次扫描即可检测所有敏感词

注意事项

  2. 自动过滤：检测到的敏感词会被替换为*号

  3. 实时检测：使用系统当前加载的敏感词列表

  4. 自动重载：系统每60秒自动从文件重新加载敏感词

敏感词检测

检测算法

系统使用三种敏感词检测算法：

  1. KMP算法（单模式匹配）

    • 适用于单个敏感词的精确匹配

    • 时间复杂度：O(n+m)

    • n为文本长度，m为敏感词长度

  2. TRIE算法（前缀树）

    • 适用于多个敏感词的快速匹配

    • 时间复杂度：O(n×m)

    • n为文本长度，m为敏感词平均长度

  3. AC自动机（推荐）

    • 适用于大量敏感词的高效匹配

    • 时间复杂度：O(n)，n为文本长度

    • 系统默认使用此算法

检测流程

  1. 用户发送聊天消息

  2. 系统使用AC自动机检测敏感词

  3. 如果检测到敏感词，拒绝消息并返回错误

  4. 如果未检测到敏感词，保存并广播消息

敏感词管理

通过REST API管理敏感词：

  • 获取所有敏感词：GET /api/sensitive-words

  • 添加敏感词：POST /api/sensitive-words

  • 删除敏感词：DELETE /api/sensitive-words/{word}

  • 批量导入：POST /api/sensitive-words/batch

  • 检测文本：POST /api/sensitive-words/check

  • 过滤文本：POST /api/sensitive-words/filter

敏感词存储

  • 敏感词存储在文本文件中，文件位置：{工作目录}/sensitive_words.txt

  • 每行一个敏感词，使用UTF-8编码

  • 系统每60秒自动从文件重新加载敏感词，修改文件后无需手动重载

注意事项

  1. 连接认证: 必须使用Token认证方式

  2. 房间管理: 用户必须先加入房间才能发送聊天消息

  3. 权限控制: 管理员功能（CREATE_ROOM, DELETE_ROOM, KICK, MUTE, UNMUTE, ADMIN）仅管理员可用

  4. 消息撤回: 只能撤回自己发送的消息，且消息发送时间不超过2分钟

  5. 私有房间: 加入私有房间需要提供正确的密码

  6. 连接管理: 连接断开后需要重新连接，会话状态不会保留

  7. 消息格式: 所有消息必须是有效的JSON格式

  8. 时间戳: 建议在发送消息时包含 timestamp 字段

  9. 错误处理: 始终监听 ERROR 类型的消息以处理错误情况

  10. 心跳保活: 建议实现心跳机制以保持连接活跃

  11. 敏感词检测: 所有聊天消息都会经过敏感词检测，包含敏感词的消息会被拒绝

  12. 历史消息: 加入房间后会收到最近10条历史消息

  13. ID格式: 所有ID为纯数字字符串，使用UUID生成的数字ID

  14. WebSocket敏感词检测: 支持通过WebSocket协议进行敏感词检测（CHECK_SENSITIVE和FILTER_SENSITIVE消息类型），功能与REST API一致

  15. 敏感词自动重载: 系统每60秒自动从文件重新加载敏感词，修改文件后无需手动重载
