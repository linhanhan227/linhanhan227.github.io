ws-room API 文档

基于源码实现 · HTTP REST 接口 · ws://localhost:8080/ws/chat 另见 WebSocket 文档

🌐 服务默认地址： http://localhost:8080
📡 REST 基础路径： /api
📦 数据格式： application/json

1. 基本信息

本文档仅描述 HTTP REST API。WebSocket 协议请参考仓库根目录下的 WEBSOCKET_DOCUMENTATION.md。

2. 鉴权与权限模型

2.1 Token 获取

通过登录接口获取 JWT： POST /api/auth/login

2.2 Token 传递方式

Authorization: Bearer <token>

2.3 权限控制规则

  • 仅被 @AdminRequired 标注的方法会被 AuthInterceptor 拦截并校验管理员权限。
  • 已配置拦截路径（WebMvcConfig）：
    • /api/admin/**
    • /api/users/*/admin
    • /api/users/*/mute
    • /api/users/*/unmute
  • 其他接口（例如用户更新自己资料）虽然读取了 request.userId，但不会自动拦截认证，调用方需要自行保证请求链路已注入该属性。

3. 响应格式说明

3.1 成功响应

{
  "message": "操作成功",
  "...": "业务字段"
}

3.2 错误响应（当前项目存在两类）

1) 统一异常处理器返回：

{
  "errorCode": "AUTH_001",
  "errorMessage": "认证失败",
  "errorCategory": "AUTHENTICATION_ERROR",
  "errorDetails": "详细信息",
  "path": "/api/auth/validate",
  "timestamp": "2026-03-09T10:00:00"
}

2) 控制器内部 try/catch 直接返回：

{
  "error": "具体错误信息"
}

3.3 常见 HTTP 状态码

状态码说明
200 OK查询/更新成功
201 Created创建成功（如用户注册、创建房间）
400 Bad Request参数错误/业务校验失败
401 Unauthorized未登录或 Token 无效
403 Forbidden权限不足（管理员接口）
404 Not Found资源不存在
405 Method Not Allowed请求方法不支持

5. 认证模块 /api/auth

方法路径鉴权说明
POST/api/auth/login否用户登录
GET/api/auth/login否登录用法提示（返回 405）
POST/api/auth/validateBearer Token校验 Token
POST/api/auth/logoutBearer Token退出登录

13. 与源码一致性的说明

  1. 文档按当前代码行为编写，不是理想化 OpenAPI 契约。
  2. 部分接口（尤其用户“仅本人可修改”的接口）依赖 request 中的 userId 属性，若未经过注入链路可能返回 403。
  3. 错误响应目前存在两套格式（error 与 errorCode/errorMessage），调用方需兼容。
  4. WebSocket 消息类型与字段较多，请以专门文档 WEBSOCKET_DOCUMENTATION.md 为准。
