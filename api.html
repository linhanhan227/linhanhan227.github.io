WebSocket 接口

连接地址

xxxxxxxxxx
ws://localhost:8080/ws/{token}

说明:

  • 使用JWT Token进行鉴权，Token与API登录获取的Token一致

  • Token有效期内可连接，Token过期或被其他设备登录会使连接断开

  • 支持多设备同时在线，每个设备独立会话

连接流程

  1. 用户通过API登录获取Token

  2. 使用Token建立WebSocket连接: ws://host:port/ws/{token}

  3. 服务端验证Token有效性

  4. 验证通过后建立连接，推送未送达消息，返回连接成功消息

  5. 验证失败则断开连接

   FAILED(3) [重试3次后]

消息可靠性保证

  1. 消息持久化: 所有通知消息先保存到数据库再推送

  2. ACK确认机制: 客户端收到消息后发送ACK确认

  3. 离线消息推送: 用户上线时自动推送未送达的消息

  4. 消息重试: 发送失败的消息最多重试3次

离线消息

  • 用户离线时，通知消息会保存到数据库

  • 用户重新连接时，自动推送所有PENDING状态的消息

  • 消息送达并收到ACK后标记为DELIVERED状态

错误情况

  • Token无效: 连接被拒绝，返回错误原因

  • Token过期: 连接被拒绝，提示Token已过期

  • Token已被其他设备登录: 连接被拒绝，提示Token已失效

  • 连接超时: 建议客户端实现心跳保活机制

最佳实践

  1. 心跳保活: 客户端每30秒发送一次ping消息

  2. 消息确认: 收到通知消息后立即发送ACK

  3. 断线重连: 连接断开后应实现自动重连机制

  4. 消息去重: 根据messageId进行消息去重处理
